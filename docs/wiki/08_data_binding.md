# 08. Data Binding Architecture

MetaUI leverages the native SAP UI5 binding engine under the hood. Because every single control generated by MetaUI is a standard native UI5 control (`sap.m.Input`, `sap.m.Switch`, etc.), the bindings work exactly as they do in standard UI5 architectures.

This document details the 4 primary binding paradigms you can use with MetaUI depending on your application's architecture.

---

## 1. OData Context Binding (Direct Backend Updates)

If your application relies on an SAP OData service, you do not need to manually parse or map JSON objects. If you bind the `DynamicHost` to a specific OData Entity Context (e.g., an `Equipment` record), all the generated fields will automatically two-way bind to those exact OData properties natively via `{meta>propertyName}`.

As the user types into the form or toggles switches, the OData model is instantly and silently updated in the background.

```xml
<!-- Your XML View -->
<meta:DynamicHost id="myODataHost" schemaDefinition="{viewModel>/equipmentSchema}" />
```

```javascript
// In your controller: Bind the host directly to an OData context path
var oHost = this.byId("myODataHost");
oHost.bindElement({
    path: "/Equipments('10000001')",
    model: "myODataModel"
});

// When the user is finished, simply submit the OData changes to the backend
onSubmitPress: function() {
    this.getView().getModel("myODataModel").submitChanges();
}
```

---

## 2. Standard Field Bindings (Two-Way JSON)

If you are not using OData, MetaUI creates an internal `sap.ui.model.json.JSONModel` (named `"meta"`) and attaches it to the `DynamicHost`.

If you provide an `inputData` object, MetaUI binds all fields to it. As the user edits the UI, the underlying JavaScript object you originally provided is mutated automatically in real-time on blur/enter.

```javascript
const myData = {
    FirstName: "John",
    IsActive: true
};

const host = new DynamicHost({
    schemaDefinition: mySchema,
    inputData: myData // 'myData' object will now be natively bound to the UI fields
});
```

---

## 3. Event-Driven Payload Extraction (On Submit)

If you prefer an event-driven architecture and don't want to worry about managing UI5 models at all, MetaUI handles the heavy lifting for you.

When a user clicks a built-in "Submit" action button (or you manually call `host.triggerSubmit()`), the engine runs all validations. If everything passes, it extracts the final, fully-constructed JSON payload from the internal model and passes it back to you via the `submit` event:

```javascript
host.attachSubmit(function(oEvent) {
    const finalJsonPayload = oEvent.getParameter("payload");
    
    // You can now send this clean, validated JSON to a REST API!
    fetch("/api/save", {
        method: "POST",
        body: JSON.stringify(finalJsonPayload)
    });
});
```

---

## 4. Live Data Telemetry (inputDataJson, outputDataJson, & liveUpdate)

If you want an external model to strictly synchronize with the form's state *without* having to write JavaScript event handlers (like in Option 3) or pass complex Javascript Objects around, MetaUI provides real-time string extraction properties.

The `DynamicHost` exposes properties that parse and stringify JSON on the fly:
- `inputDataJson`: Natively accepts a raw JSON string. If this string is mutated externally after generation, MetaUI instantly hot-swaps the UI controls without redrawing the layout.
- `outputDataJson`: The stringified JSON representation of the payload. Automatically updates when a field loses focus (native UI5 behavior).
- `liveUpdate="true"`: If enabled, pushes data to `outputDataJson` on *every single keystroke*.
- `isValid`: A read-only boolean you can bind to your Save button to instantly disable it if a field fails validation.

### Flawless 2-Way Sync (The "Echo Chamber")
You can safely bind `inputData` and `outputData` to the exact same shared UI5 model property. 

To prevent catastrophic infinite rendering loops and cursor-jumping (where UI5 echoes the new data back down into the control), MetaUI features an advanced internal Equality Check.
- **Deep Equality:** The engine uses native `sap/base/util/deepEqual` to compare incoming payloads against the last known output, completely eliminating performance bottlenecks and key-ordering fragility.
- **Focus Preservation:** When hot-swapping data, the Engine explicitly passes `suppressInvalidate: true` to the UI5 framework. This guarantees that the RenderManager will not destroy the DOM of your form, perfectly preserving the user's keyboard focus mid-keystroke.

```xml
<mvc:View
    xmlns:mvc="sap.ui.core.mvc"
    xmlns:meta="nz.co.siliconst.ui5.metaui.controls"
    xmlns="sap.m">
    
    <!-- 
      We pass data into inputDataJson, and extract it via outputDataJson.
      We enable liveUpdate="true" to get a keystroke-by-keystroke stream.
    -->
    <meta:DynamicHost 
        schemaDefinition="{viewModel>/parsedSchema}" 
        inputDataJson="{viewModel>/liveFormString}" 
        outputDataJson="{viewModel>/liveFormString}"
        liveUpdate="true"
        debugMode="true" />
        
    <!-- A side-by-side view that updates instantly as the user types in the form. -->
    <!-- If you edit this TextArea directly, the form above will also instantly update! -->
    <TextArea 
        value="{viewModel>/liveFormString}" 
        editable="true" 
        rows="20" />
        
</mvc:View>
```
